Implementation Details - Phase 1

üìä Phase 1: Data Preparation & Indexing

Step 1: Document Loading
  • Medical PDFs stored in data/ folder
  • LangChain's PyPDFLoader extracts text content
  • Handles multiple documents automatically
Step 2: Text Chunking
  • RecursiveCharacterTextSplitter breaks documents into manageable chunks
  • Maintains context between chunks for better retrieval
  • Optimized chunk size for embedding model
Step 3: Embedding Generation
  • HuggingFace model: sentence-transformers/all-MiniLM-L6-v2
  • Converts text to 384-dimensional vectors
  • Captures semantic meaning for similarity search
Step 4: Vector Storage
  • Upload embeddings to Pinecone vector database
  • Create searchable index for fast retrieval
  • Execute: python store_index.py

Implementation Details - Phase 2

‚ö° Phase 2: Application Runtime

User Interaction Flow
  • Interface: ChatGPT-style web UI built with Flask
  • User Input: Natural language medical questions
  • Real-time processing: Immediate response generation
Query Processing Pipeline
  • Step 1: User question embedded using same HuggingFace model
  • Step 2: Semantic search in Pinecone retrieves relevant medical context
  • Step 3: LangChain combines query + retrieved context into optimized prompt
  • Step 4: OpenAI GPT generates contextual response
  • Step 5: Response displayed in chat interface
üöÄ Launch: python app.py ‚Üí Access at localhost:8080

Implementation Details - Phase 3

‚òÅÔ∏è Phase 3: AWS Deployment (CI/CD)

1. Code Push ‚Üí GitHub Actions Trigger
  • Developer commits code to main branch
  • GitHub Actions workflow automatically starts
2. Docker Containerization
  • Workflow builds Docker image from Dockerfile
  • Includes all dependencies and application code
  • Creates portable, consistent environment
3. Push to AWS ECR
  • Docker image uploaded to Elastic Container Registry
  • Stored securely in AWS cloud
  • ECR URI: 970547337635.dkr.ecr.ap-south-1.amazonaws.com/medicalchatbot
4. Deploy to EC2
  • Self-hosted GitHub Actions runner on EC2 instance
  • Automatically pulls latest image from ECR
  • Runs Docker container ‚Üí Application live!
